## Current Language Rule

The current active product surface only needs to support:
- Simplified Chinese (`zh-CN`)

Traditional Chinese is not required in the current stage.

Rules:
1. Do not hardcode English in active product surfaces unless explicitly required.
2. All active user-facing product copy should be Simplified Chinese first.
3. AI outputs should default to Simplified Chinese.
4. Do not spend sprint scope maintaining Traditional Chinese in this phase.

## Homepage Rule

The homepage is not a brand-story page.
It is a task-entry page.

Users arriving at Tideus already know they want to do something.
The homepage must prioritize immediate product actions over explanation.

The homepage hero must focus on:
- Ask AI
- Start a Case / Start Tracking
- Open / Continue Workspace
- Supported scenarios:
  - Visitor Record
  - Study Permit Extension

The homepage must avoid:
- long brand storytelling
- long workflow philosophy blocks
- excessive product explanation
- unnecessary copy above the fold

## Public Surface Rule

Public-facing surfaces should remain short, structured, and action-oriented.

### Homepage
Must emphasize:
- immediate entry
- core outputs
- scenario shortcuts
- clear CTA

### Scenario pages
Must explain:
- what this scenario is
- who it is for
- how AI helps
- what output the user gets
- direct action CTA

Do not turn scenario pages into long essays.

### AI answer surface
Must remain task-oriented, not generic chat-like.

Visible output should prioritize:
- answer summary
- why this matters
- relevant public guidance note
- next steps
- action buttons:
  - Save to workspace
  - Generate checklist
  - Start tracking
  - Continue in case workspace

### Workspace
Must remain the core operational surface, clearly showing:
- Case / Task header
- Current status
- Materials summary
- Latest review summary
- Missing items
- Risk flags
- Next recommended action
- Review history
- Export / Handoff
- Ask AI about this case
